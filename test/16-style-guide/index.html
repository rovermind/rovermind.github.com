Markup Style Guide

HTML/CSS 코딩 스타일 가이드라인(Guideline)

기본 규칙

  • HTML은 해당 DTD의 명세에 맞게 작성하며, W3C validation을 통과해야 한다.
  • 기본 DTD는 HTML5로 선언한다. DTD를 제외한 모든 요소와 애트리뷰트는 소문자로 작성한다.
    <!DOCTYPE html>
      ...
    잘못된 예 올바른 예
    <SPAN Class="desc">간단한 설명</SPAN> <span class="desc">간단한 설명</span>
  • 기본 인코딩은 utf-8을 원칙으로 한다. utf-8 인코딩이 어려우면 euc-kr을 사용한다.
    <meta charset="utf-8">
      ...
    <meta http-equiv="Content-Type" content="text/html;charset=euc-kr">
      ...
  • 애트리뷰트값은 큰따옴표(" ")로 묶는다.
    잘못된 예 올바른 예
    <img src='test.gif' width='100' height='100' alt='테스트'> <img src="test.gif" width="100" height="100" alt="테스트">
  • 특수 기호는 문자 엔티티 참조를 사용하여 코드로 변환한다.
    잘못된 예 올바른 예
    <a href="...&nid=2">Q&<A> <a href="... &amp;nid=2">Q&amp;A<a>

네이밍 규칙

  • 네이밍 규칙은 레이아웃, 객체, 이미지, 폴더, 파일의 이름을 작성하는 규칙으로 이해하기 쉬운 이름으로 작성해야 한다.
  • 네이밍은 영문 소문자, 숫자, 대시(-), 언더스코어(_)로 작성한다. 이 때 언더스코어는 이미지에만 사용하고, 나머지는 대시로 구분한다.
    기본형 잘못된 예 올바른 예
    section sectionList section-list
  • 네이밍의 시작은 영문 소문자만 가능하다.
    기본형 잘못된 예 올바른 예
    aside Left-aside left-aside
  • 네이밍은 가능하면 축약하지 않는다.
    기본형 잘못된 예 올바른 예
    button btn-close button-close

ID와 클래스 네이밍 규칙

  • 한 문서에서 동일한 ID를 여러 번 사용하지 않는다.
  • 레이아웃을 제외한 ID는 스타일을 지정하지 않는다.
  • 클래스는 여러 번 사용할 수 있다.

이미지 네이밍 규칙

  • 이미지 네이밍은 '형태_의미_상태'의 순서로 조합한다. 이름은 언더스코어로 구분한다.
  • 같은 분류의 이미지가 두 개 이상이면 마지막에 숫자를 추가하여 구분한다.
  • 임시 이미지는 'tmp_'를 조합한다.
잘못된 예 올바른 예 설명
on_recommend_tab1 tab1_recomm_on '형태_의미_상태' 순서로 조합한다.
bnm.gif btn_naver_mail.gif 임의로 축약하지 않는다.
btn_Search.gif btn_srch.gif 영문 소문자를 사용한다
1btn_search.gif btn_srch.gif 숫자로 시작하지 않는다

HTML 코드 작성 규칙

들여쓰기

HTML 코드를 작성할 때 코드의 가독성을 높이기 위하여 왼쪽 첫 번째 열부터 오른쪽으로 일정한 간격(tap 2칸 / space 4칸)만큼 띄어 쓴다.

<table>     <caption>...</caption>     <colgroup>...</colgroup>         <col>...</col>     <colgroup>...</colgroup>     <thead>         <tr>            <th>...</th>            <th>...</th>         </tr>     </thead>   ...

주석

HTML 코드의 주석은 코드 그룹을 구분하거나, 참고해야 하는 사항을 기술한다.

<!-- content --> <div id="content"> <!-- 네임카드 --> <div class="namecard"> ... </div> <!-- //네임카드 --> </div> <!-- //content -->   ...

CSS 코드 작성 규칙

들여쓰기

CSS 코드는 들여쓰기를 하지 않는다. 단, 중괄호"{}"가 중첩되는 경우 예외

잘못된 예 올바른 예
#content{border:1px solid #000}
#content .class{color:#000}		 #content{border:1px solid #000}
#content .class{color:#000}
@media all and (min-
width:480px){.vod_wrp2{height:278px}}		 @media all and (min-width:480px){
    .vod_wrp2{height:278px}
}

공백

CSS 코드는 공백을 최소화한다

빈 줄

CSS 코드의 빈 줄은 코드 그룹의 영역을 표시하기 위하여 사용한다.

선택자

  • 공통 선택자(*) 사용은 지양한다.
  • ID 선택자는 레이아웃(wrap, header, container, content, footer)에만 사용한다.
  • ID와 클래스 간의 다중 선택자를 사용하지 않는다.
잘못된 예 올바른 예
class.bg_v1{background:green} .bg_v1{background:green}

속성 선언 순서

레이아웃과 관련이 큰 것에서 시작하여 레이아웃과 무관한 것 순서로 선언한다.

순서 의미 대표되는 속성
1 레이아웃 display, visibility, overflow, float, clear, position, top, right, bottom, left, zindex
2 BOX width, height, margin, padding, border
3 배경 background
4 폰트 font,color, letter-spacing, text-align, text-decoration, text-indent, verticalalign, white-space
5 레이아웃 위에 언급되지 않은 나머지 속성들로 폰트의 관련 속성 이후에 선언하며, 기타 속성 내의 선언 순서는 무관함

z-index

z-index 속성의 속성 값의 범위는 최소 10, 최고 1000이며, 10 단위로 증감한다. 단, 10 단위 사이의 예외 변수가 발생하면 1 단위 값을 지정할 수 있다

핵(hack)

핵은 가능한 한 사용하지 않는다.

웹표준 기반의 마크업

  • W3C 표준에 근거한 마크업은 향후 웹 브라우저 호환성을 보장받을 수 있다.
  • 의미에 맞는 HTMl 요소를 사용한 마크업은 HTML 요소의 의미만으로 구조화, 선형화하여 정보를 전달할 수 있다.
  • 구조와 표현을 분리한 마크업은 문서의 정보는 HTML로, 디자인은 CSS로 표현할 수 있다. 이는 쉬운 유지보수를 가능케 하며, 로딩 시간을 단축하고 HTML 소스 코드의 재사용성이 높아지는 장점이 있다.
